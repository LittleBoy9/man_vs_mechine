Manifesto / Principles

Against Frictionless Thought

The site name stays rough on purpose. “Mechine” is left visible like a typo in a photocopied zine: a reminder that polish is not the same thing as honesty.

Core rules

What this project argues for

  • Readers should be able to sense authorship.
  • Automation should reduce labor, not erase accountability.
  • Images can be decorative, but here they also carry language.
  • Design should make claims visible, not just attractive.
01

Use tools, keep signatures

Machine assistance is acceptable when it remains part of an auditable workflow rather than pretending to be neutral magic.

02

Prefer visible seams

A site about authorship should show notes, labels, captions, and rough edges instead of hiding every sign of process.

03

Put text inside visuals

The posters and banners on this site are not filler. They behave like arguments, with headlines, subheads, and short editorial copy.
